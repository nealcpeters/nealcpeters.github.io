Error Messages and Life

May 27th, 2014

Every programmer encounters error messages. There may be a difference in quantity as a developer grows more experienced, but the fact of the matter is that errors and programming are intertwined.

A beginning developer gets their first error message and likely grows frustrated. When you’re learning to code, just learning the syntax and approach can be difficult enough. Until you have a reasonable grasp on those concepts, it can be very difficult to interpret what the error message even means. Most new developers don’t even read the error message for this reason.

As developers grow and become more experienced, they start to read the error messages. When code isn’t running, reading and interpreting the error message is the first clue as to what’s broken. The insight provided by error messages often leads to solved problems.

I’m not sure that anyone ever truly gets excited over receiving an error message, but there is and should be a stark difference between approaches. Inexperienced developers view error messages as cause to flip tables or grow in irritation, possibly even give up. Skilled and experience developers know that error messages are the keys to improvement. They view an error as feedback, telling them what they did wrong and how to fix it.

As my experience with coding grows, I view reading error messages not as a sign of indiscretion, but as part of the process in building a successful and functioning piece of software. I’ve noticed that this approach to error messages has worked its way into the rest of my life.

All of life’s failures and setbacks should be viewed the same as an experienced programmer approaches error messages. Failures should offer insight into what went wrong, which leads to a clear area which necessarily needs improvement.

Did you not get the job because you didn’t know the answer to an architecture question? Did your significant other leave you because you never communicated? Obviously either scenario isn’t fun and is a little more serious than an error message in the terminal, but the principle in how one reacts should be the same. One should be able to clearly define what went wrong, and what needs to be done in order to prevent the same in the future.

Your life is an application, and a setback is only an error message offering clues for successful navigation.
